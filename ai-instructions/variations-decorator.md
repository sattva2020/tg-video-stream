# Промпт-декоратор для команды +++variations(number, temperature)

## Системная инструкция

Когда пользователь использует команду `+++variations(number, temperature)`, активируется специальный режим создания вариаций дизайна HTML файла.

### Параметры команды:

- `number` (1-10) - количество вариаций для создания
- `temperature` (0.1-2.0) - степень креативности:
  - 0.1-0.5: Минимальные изменения (цвета, шрифты, отступы)
  - 0.6-1.0: Умеренные изменения (макет, компоненты, стили)
  - 1.1-1.5: Значительные изменения (структура, концепция)
  - 1.6-2.0: Радикальные изменения (полная переработка дизайна)

### Обязательные действия:

1. **Анализ исходного файла:**

   - Изучить структуру HTML
   - Определить основные дизайн-элементы
   - Выявить цветовую схему и типографику
   - Понять назначение и функциональность

2. **Обработка референсов (если приложены):**

   - Извлечь дизайн-идеи из референсных изображений
   - Адаптировать стили под концепцию оригинала
   - Использовать элементы вдохновения соответственно температуре

3. **Создание вариаций согласно температуре:**

   **Низкая температура (0.1-0.5):**

   - Изменение цветовой палитры
   - Корректировка шрифтов и размеров
   - Незначительные изменения отступов и размеров
   - Сохранение общей структуры

   **Средняя температура (0.6-1.0):**

   - Реорганизация блоков контента
   - Изменение стилей компонентов
   - Добавление новых UI элементов
   - Эксперименты с анимациями

   **Высокая температура (1.1-1.5):**

   - Кардинальное изменение макета
   - Новые концептуальные подходы
   - Альтернативные способы навигации
   - Инновационные визуальные решения

   **Максимальная температура (1.6-2.0):**

   - Полная переработка дизайн-концепции
   - Экспериментальные подходы
   - Нестандартные интерактивные элементы
   - Революционные визуальные решения

4. **Структура файлов:**

   ```
   [original-file-directory]/
   ├── original-file.html
   └── variations/
       ├── original-file - variation 1/
       │   ├── index.html
       │   └── assets/ (если нужно)
       ├── original-file - variation 2/
       │   ├── index.html
       │   └── assets/ (если нужно)
       └── ...
   ```

5. **Требования к каждой вариации:**

   - Полностью функциональный HTML файл
   - Встроенные CSS стили (или отдельные файлы в папке assets)
   - Сохранение основного функционала оригинала
   - Уникальность каждой вариации
   - Соответствие заданному уровню температуры

6. **Документация изменений:**
   - Создать файл `README.md` в папке variations
   - Описать концепцию каждой вариации
   - Указать ключевые изменения и особенности
   - Добавить информацию о температуре и подходе

### Пример использования:

```
+++variations(5, 1.2)
[прикрепленный HTML файл]
[референсные изображения - опционально]
[дополнительные требования - опционально]
```

### Обязательный вывод:

1. Краткое описание проанализированного оригинала
2. Объяснение стратегии для заданной температуры
3. Создание всех файлов вариаций
4. Краткое описание каждой созданной вариации
5. Инструкции по просмотру результатов

### Ограничения:

- Максимум 10 вариаций за одну команду
- Температура не может быть выше 2.0
- Сохранение базовой функциональности оригинала
- Все файлы должны быть валидными и рабочими

---

**Активация:** Команда активируется только при точном написании `+++variations(number, temperature)` в начале сообщения пользователя.
