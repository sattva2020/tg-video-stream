# Регламент копирайта лендинга

> Версия 2025-12-04 · Автор: Jarvis

Документ фиксирует развернутые формулировки для каждого сектора лендинга. На страницах оставляем только главный тезис и 3 буллета; остальные детали берём отсюда или выносим в FAQ.

## Hero (Telegram Streamer)
- **Контекст:** 24/7 вещание в Telegram без OBS и кастомного железа.
- **Месседж:** "Плейлисты, фейловеры и расписания запускаются одной консолью; команда работает по регламенту, а не тушит пожары."
- **Для кого:** владельцы каналов, продюсеры/стримеры, админы комьюнити, новостные редакции.
- **Что получает пользователь:** готовые ротационные плейлисты, fail-safe раннер с проактивными health-check, переключение <100 мс, отсутствие babysitting OBS.
- **Чёткое объяснение продукта:** Telegram Streamer позволяет запускать эфиры без OBS, миксовать YouTube/MP4/RTMP, моментально переключать сегменты, восстанавливаться после сбоев и планировать эфиры с учётом часовых поясов.
- **Визуал:** заголовок с явным outer glow и размером +1–2 шага относительно базовой сетки.

## Feature Section
- **Тезис:** «Трансляции в Telegram как регламентируемый плейбук» → раскрывается короткими абзацами и тремя буллетами.
- **Группы:**
  - **Базовые:** ingest, источники, наблюдаемость.
  - **Продвинутые:** расписания, безопасность, операции.
  - **Уникальные:** мультиканальность, self-healing, enterprise SLA.
- **UI:** карточки с цветовым кодом по группам, иконка Lucide, компактное описание (≤2 bullet'ов).
- **Метрики:** 3 бара (аптайм, auto-recovery, approvals) с подсказками.
- **CTA:** после секции повторяем «Начать работу» с коротким пояснением "Снимите ingest и мониторинг за минуты".

## Workflow Section
- **Тезис:** "Каждый этап снимает неопределённость до выхода в эфир".
- **Компактный чек-лист:** 4 коротких шага "Подключите аккаунт → Добавьте контент → Настройте недельный план → Нажмите старт".
- **Карточки:** развивают каждый шаг (авторизация, плейлисты, расписание, мониторинг) и содержат минимум копирайта.
- **CTA повтор:** после секции — "Готовы запускать? Войдите и стартуйте".

## Use Cases Section
- **Контекст:** показать три типичных сценария телеграм-вещания — новостные редакции, 24/7 радио/стримы и медиахабы.
- **Overline:** «Примеры использования».
- **Заголовок:** «Где уже работает Telegram Streamer».
- **Подзаг:** «Для редакций, радиостанций и медиахабов».
- **Карточки:**
  - **Новостные каналы:** breaking news, экстренные обращения и марафоны без пауз.
  - **24/7 радио и стримы:** музыка и подкасты продолжаются без дежурных у OBS.
  - **Автотрансляции медиа:** повтор ТВ/архивов по расписанию, с fail-safe раннерами.
- **UI:** карточки с мягким градиентом (sky/violet/amber), без кнопок; текст ≤3 строки.
- **Цель:** описать бизнес-ценность и показать, что продукт уже работает для реальных сегментов.
