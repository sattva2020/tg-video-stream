# Frontend Requirements Checklist: Современный лендинг Telegram 24/7 Video Streamer

**Purpose**: Проверка качества требований для фронтенд-лендинга с приоритетом на нефункциональные метрики перед PR-ревью
**Created**: 2025-11-22
**Feature**: [spec.md](../spec.md)

## Requirement Completeness

- [x] CHK001 Описаны ли все обязательные элементы hero-блока (заголовок, подзаголовок, визуальный слой, CTA) и их положение на первом экране? [Completeness, Spec §FR-001]
- [x] CHK002 Указаны ли явные требования к единственной кнопке «Вход», включая маршрут `/login`, aria-атрибуты и размеры hit-area? [Completeness, Spec §FR-002]
- [x] CHK003 Зафиксирован ли полный состав локализованных текстов (заголовки, преимущества, подписи CTA) для всех поддерживаемых языков и fallback на en? [Completeness, Spec §FR-003, §FR-008]

## Requirement Clarity

- [x] CHK004 Заданы ли числовые значения для gradient/blur токенов и фирменных цветов, чтобы визуальные ревьюеры могли однозначно сверить стили? [Clarity, Spec §FR-004, Plan §Phase 1 T002]
- [x] CHK005 Чётко ли сформулированы условия переключения между WebGL, видеослоем и статичным постером, включая детект `prefers-reduced-motion`? [Clarity, Spec §FR-005, Spec §Edge Cases]
- [x] CHK006 Содержит ли требование по клавиатурному пути до CTA конкретное ограничение (≤3 табов) и описание, какие элементы могут встречаться между фокусами? [Clarity, Spec §FR-007]

## Requirement Consistency

- [x] CHK007 Совпадают ли упоминания маршрута `/login` и поведения CTA во всех документах (Spec §FR-002, Tasks §Phase 3 T008/T010), без альтернативных ссылок на другие экраны? [Consistency]
- [x] CHK008 Не противоречат ли друг другу требования по количеству преимуществ (Spec §FR-008) и задачи на их реализацию (Tasks §T031/T032), включая локализацию и порядок отображения? [Consistency]

## Acceptance Criteria Quality

- [x] CHK009 Связаны ли сценарии успеха SC-001…SC-004 с измеримыми метриками и соответствующими тестовыми задачами (например, T011, T015, T021), чтобы ревьюер мог проследить выполненность? [Acceptance Criteria, Spec §SC]
- [x] CHK010 Предусмотрено ли сопоставление NF метрик (fps, TTI, layout shift) с конкретными инструментами (Playwright, Lighthouse), чтобы результаты можно было объективно подтвердить? [Acceptance Criteria, Spec §NF-001…NF-003, Tasks §T026/T037]

## Scenario Coverage

- [x] CHK011 Описаны ли сценарии ручного переключения языка, включая сохранение фокуса и немедленное обновление текста без перезагрузки? [Coverage, Spec §User Story 2, Spec §FR-003]
- [x] CHK012 Есть ли требования к отображению списка преимуществ при нулевых или частично отсутствующих переводах, чтобы избежать пустых блоков? [Coverage, Spec §FR-008, Tasks §T031, Gap]

## Edge Case Coverage

- [x] CHK013 Задокументированы ли поведения при ширине <320 px (до 280 px), включая отсутствие горизонтальной прокрутки и сохранение CTA в viewport? [Edge Case, Spec §Edge Cases, Tasks §T021]
- [x] CHK014 Описан ли fallback для неподдерживаемых языков (es → en) вместе с подсказкой в переключателе? [Edge Case, Spec §Edge Cases, Tasks §T024]
- [x] CHK015 Определено ли, что происходит, если загрузка WebGL/видео превышает 2 секунды, и как skeleton/градиент должны вести себя визуально? [Edge Case, Spec §Edge Cases, Tasks §T025]

## Non-Functional Requirements

- [x] CHK016 Указаны ли пороговые значения для TTI и fps (≤2 c, ≥45 fps) и методика снятия метрик (landing-lh.mjs, landing-visual-metrics.spec.ts) с ссылкой на места фиксации результатов? [Non-Functional, Spec §NF-001/§NF-002, Tasks §T026/T037]
- [x] CHK017 Описано ли требование по поведению при отсутствии пользовательских шрифтов, включая стек `system-ui` и допустимый layout shift ≤2%? [Non-Functional, Spec §NF-003, Tasks §T033/T034]
- [x] CHK018 Зафиксированы ли требования по контрасту (WCAG AA) и клавиатурной доступности вместе со способами проверки (axe-core сценарии)? [Non-Functional, Spec §FR-007, Spec §NF-004, Tasks §T022/T023]

## Dependencies & Assumptions

- [x] CHK019 Прописаны ли зависимости от существующей i18n-инфраструктуры (frontend/src/i18n.ts) и требования к обновлению bundles до начала разработки? [Dependency, Spec §FR-003, Tasks §T007/T012]
- [x] CHK020 Уточнено ли, что ZenScene или альтернативная 3D сцена уже доступна в проекте, либо требуется новая реализация, чтобы избежать скрытых предположений? [Assumption, Spec §FR-004, Plan §Phase 5]

## Ambiguities & Conflicts

- [x] CHK021 Разъяснены ли термины «иммерсивный фон» и «фирменный субстрат» в измеримых характеристиках (яркость, интенсивность, частота анимации), чтобы исключить субъективность на ревью? [Ambiguity, Spec §FR-004/§FR-005]
- [x] CHK022 Исключают ли требования конфликт между WebGL-анимацией и CTA (например, нет перекрытия фокусом или кликами) и где задокументированы соответствующие ограничения? [Conflict, Spec §User Story 3, Tasks §T016/T018]
